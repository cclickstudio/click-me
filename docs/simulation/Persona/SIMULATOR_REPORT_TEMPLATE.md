# ClickMe 시뮬레이터 — 광고 분석 보고서 출력 양식 v1.1

> **이 문서의 목적**
> 시뮬레이터가 사용자(마케터)에게 최종 출력하는 분석 보고서의 표준 양식을 정의한다.
> 구조 설계 근거: 실무 광고 사전테스트 보고서 관행 조사 (System1 Test Your Ad,
> Qualtrics Ad Testing Program, AdTest.AI, 진단형 카피테스팅 문헌).
>
> **v1.1 (2026-06-11) 변경:** ① §0 입력 필드 확장(선언 의도·상품 분류·패널 버전) ② §1 "예측 CTR" → "시뮬레이션 클릭 의향률" (보정 전 실측 스케일 출력 금지) ③ §2-3 거부 사유 태그를 구조화 출력 방식으로 정정 ④ §6 도입부 "4-A/4-B" 오타 → "6-A/6-B" ⑤ 데이터 모델 매핑을 ANALYSIS_AGENT_STRATEGY(토론·루브릭 패스)와 정합화.
>
> **설계 원칙 4가지**
> 1. **결정 → 근거 → 진단 → 행동** 순서. 마케터는 위에서부터 읽다가 필요한 깊이에서 멈춘다.
> 2. **모든 점수에 비교 기준을 붙인다.** 맨숫자(naked number)는 의미가 없다. (System1 방식: 항상 카테고리 평균 병기)
> 3. **"무엇"보다 "왜"를 보여준다.** 점수는 선택을 돕고, 사유는 개선을 돕는다.
> 4. **예측은 레인지로, 사실은 출처와 함께.** 단언 금지 — "직감보다 나은 근거" 포지셔닝 유지.
>
> **보고서의 세 독자**
> | 독자 | 보는 섹션 | 원하는 것 |
> |---|---|---|
> | 결정권자 (팀장·클라이언트) | §1만 | Go / No-Go / 어느 안? |
> | 실무 마케터 | §1~§3 | 왜 그런 결과인지, 어디서 새는지 |
> | 제작자 (디자이너·카피라이터) | §4~§5 | 정확히 뭘 고치라는 건지 |

## §0. 헤더 — 테스트 개요

보고서 신뢰성의 출발점. 무엇을, 누구에게, 언제 테스트했는지 명시.

```
┌──────────────────────────────────────────────────────┐
│  📋 광고 시뮬레이션 보고서                              │
│                                                      │
│  광고명        : 여름 신제품 런칭 — A안 (영상 15s)      │
│  업종 카테고리  : 식음료 > 음료 (사용자 선언 · 감지 일치 ✅)│
│  상품/서비스   : 신제품 제로슈거 음료                    │
│  광고 목적     : 신제품 인지 + 구매 전환                │
│  테스트 일시   : 2026-07-02 14:30                      │
│  페르소나 규모  : 1,000명 (전국 분포 샘플링)            │
│  타깃 모드     : 자동 추론 (전체 인구) ⓘ               │
│  비교 대상     : B안 (보고서 #1024) — 비교 보기 →       │
│  엔진 버전     : persona-pipeline v0.3 / panel-v1 / claude-x.x │
└──────────────────────────────────────────────────────┘
```

| 필드 | 설명 | 비고 |
|---|---|---|
| 광고명 / 포맷 | 사용자 입력 + 자동 감지 (이미지/영상/카피) | |
| 업종 카테고리 · 상품/서비스 분류 | **사용자 입력(선언 의도)** + 자동 분류 교차검증 — 충돌 시 경고 표시 (ANALYSIS §3.5-2) | §6-A 벤치마크 · 처방 RAG `industry` 필터 매칭 키 + §3-1 의도 비교 기준점 |
| 광고 목적 | 인지 / 클릭 / 전환 / 브랜딩 중 선택 | 종합 점수 가중치가 목적에 따라 달라짐 |
| 페르소나 규모·분포 | 샘플링 방식 요약 | 신뢰의 근거 |
| 타깃 모드 | 자동 추론(전체 인구, **기본**) vs 사용자 지정(연령·성별) | 지정 시 Layer 1 `target_filter` 조건부 샘플링 (PERSONA 단계 1) — 최소 표본(300명) 미만이면 경고 + 신뢰도 하향 |
| 엔진 버전 | 모델 버전 pin + 패널 버전(`panel_version`) 표시 | 재현성 증빙 (PERSONA §3.6 고정 패널) |

## §1. 결론 카드 — 의사결정 섹션

**규칙: 이 섹션만 보고 Go/No-Go 결정이 가능해야 한다. 스크롤 없이 한 화면.**

```
┌──────────────────────────────────────────────────────┐
│   종합 효과 점수                                       │
│                                                      │
│        72 / 100        등급  B+                       │
│        ━━━━━━━━░░░     업종 평균 64 대비 +8 ⓘ          │
│                                                      │
│   "주목도와 첫인상은 업종 상위권이나, 행동 유도(CTA)가    │
│    약해 관심이 클릭으로 이어지지 않는 구조"              │
│                                                      │
│   ✅ 강점  강한 비주얼 훅 · 20대 여성 공감도 높음        │
│   ⚠️ 약점  CTA 불명확 · 40대 이상 거부감                │
│                                                      │
│   클릭 의향률  16 ~ 22% (95% CI · 시뮬레이션 패널) ⓘ    │
│   판정      ⚠️ 조건부 집행 권장 — §5의 개선 1·2 반영 시  │
│             A안 > B안 (B안 종합 61점)                  │
│                                                      │
│   신뢰도    ●●●○ 응답 분산 정상 / 20-30대 P1 grounding  │
└──────────────────────────────────────────────────────┘
```

### 필드 정의

| 필드 | 산출 방식 | 표시 규칙 |
|---|---|---|
| 종합 효과 점수 (0~100) | §2 반응 집계 + §4 진단 점수의 가중 합성. **가중치는 §0의 "광고 목적"에 따라 변동** (전환 목적이면 Action 가중↑) | 반드시 업종 평균 병기 |
| 등급 (S/A/B/C/D) | 점수 구간 매핑 | 클라이언트 커뮤니케이션용 |
| 한 줄 판정 | LLM 생성 — 강점·약점·구조적 원인을 한 문장에 | 30자 내외, 진단적 어조 |
| 강점/약점 Top 2 | §4 진단에서 최고·최저 항목 자동 추출 | 각 2개 이내 |
| 시뮬레이션 클릭 의향률 | QA 통과 응답 중 Action 단계 통과 비율 — **순수 코드 집계**. 레인지는 보정이 아니라 **통계적 신뢰구간**(이항 95% CI 또는 부트스트랩) | **보정 전 실측 CTR 스케일 환산·표기 금지.** 상대 비교(A/B 우위 · 업종 시뮬레이션 기준선 대비) 중심 표기 |
| 판정 | 집행 권장 / 조건부 권장 / 재제작 권장 3단계 | A/B 비교 시 상대 판정 병기 |
| 신뢰도 표시 | 응답 분산 정상성 + grounding 등급(P1/P2) 합성 | 4단계 도트. ⓘ 클릭 시 §7로 |

> **A/B 비교 모드:** 두 안 이상 테스트 시, 결론 카드가 비교 테이블로 전환된다.
> 항목별로 어느 안이 우세한지 + "B안은 40대에서, A안은 20대에서 우세" 같은
> 세그먼트 교차 판정을 추가한다. (실무 조사: "어느 안" 못지않게 "왜"가 중요)
> ※ A/B 비교는 반드시 **동일 패널**(PERSONA §3.6)에 두 안을 노출해 수행한다 — 패널 차이와 광고 차이의 교락 방지.

> **"예측 CTR"(실측 스케일)은 calibration 이후 해금되는 기능이다.**
> 보정 데이터는 광고 매니지먼트(기획서 4-2)의 실측 트래킹에서 나온다 —
> 시뮬레이션 의향률 ↔ 실제 CTR 페어 축적 → 회귀 매핑.
> 보정 전에 실측 스케일 수치를 단일이든 레인지든 출력하는 것은 §5의
> "+N점 약속 금지"와 동급 위반이다. (시뮬레이터와 매니지먼트가 한 플랫폼에
> 있어야 하는 기술적 근거 — 발표 포인트로 활용)

## §2. 페르소나 반응 집계 — 근거 섹션 ★

**ClickMe의 차별화 본체. "1,000명의 한국 가상 소비자가 이렇게 반응했다"의 시각화.**

### 2-1. AISAS 퍼널

```
  Attention   Interest    Search      Action      Share
   ████ 82%    ███ 61%    ██ 34%      █ 19%       ▌8%
                              ▲
                        ⚠️ 최대 이탈 구간 (-27%p)
                        "관심은 갔지만 더 알아볼 이유를 못 찾음"
```

| 출력 요소 | 규칙 |
|---|---|
| 단계별 통과율 | 페르소나 응답의 구조화 필드 집계 |
| 최대 이탈 구간 하이라이트 | 인접 단계 간 낙폭 최대 지점 자동 표시 + 이탈 사유 요약 (QA 통과 응답의 사유 태그 집계) |
| 업종 퍼널 평균 오버레이 | §6 벤치마크 연동 (데이터 있는 단계만) |

> **이탈 구간이 §5 개선 권고의 1순위 입력이 된다.** 퍼널은 예쁜 그림이 아니라
> "어디를 고치면 되는지"를 가리키는 화살표다.

### 2-2. 구매의도 분포 (히스토그램)

```
  전혀 없음   낮음      보통      높음     매우 높음
   ███ 24%   ██ 18%   ████ 31%  ███ 21%   ▌6%

   평균 2.67 / 5  ·  표준편차 1.21  ✅ 응답 분산 정상
```

- **평균만 보여주지 않는다. 분포 전체를 보여준다.** 분산이 정상 범위라는 표시 자체가 "동질화 없음"의 증거 (제품 신뢰성 증빙을 보고서 안에 내장).
- 분산이 비정상적으로 낮으면 보고서에 경고 배지: "⚠️ 응답 집중 — 재시뮬레이션 권장"

### 2-3. 신뢰도 & 거부 분석

```
  광고 신뢰도   3.1 / 5
  거부율       23%
  
  거부 사유 분해:
   "나와 무관한 광고"        41%  ← 타기팅 미스매치 신호
   "과장된 표현 같음"        23%  ← 카피 신뢰성 신호
   "광고라서 그냥 스킵"      19%  ← 광고 거부감 (한국 특화 지표)
   "브랜드를 모름/안 믿음"   11%
   기타                      6%
```

- 거부 사유 태그는 사전 정의 분류 체계(taxonomy enum)를 따른다 — 페르소나 반응 스키마(PERSONA §3.5)의 구조화 필드로 **LLM이 직접 enum 출력**하여 집계한다 (자유 응답을 사후 분류하지 않음 — 분류 오차 단계 제거).
- **각 사유가 §4의 어느 진단 항목과 연결되는지 화살표 주석** (예: "과장" → 카피 신뢰성).

### 2-4. 감정 반응 분포

```
  호기심 ████ 34%   공감 ██ 22%   무관심 ██ 24%   불신 █ 12%   거부감 ▌8%
  
  지배 감정: 호기심 — "새로운 건 알겠는데, 그래서 뭐가 좋은데?"
```

- 지배 감정 + 그 감정의 대표 발화 1개를 병기 (숫자에 얼굴을 붙인다).

## §3. 타깃 인사이트 — "누구에게 먹히나"

### 3-1. 세그먼트 히트맵

```
              종합   Attention  Action   신뢰도
  20대 여성    81      91        31       3.8     ★ 최고 반응
  20대 남성    68      77        22       3.2
  30대 여성    74      83        24       3.4
  30대 남성    63      71        17       3.0
  40대 이상    52      58        11       2.5     ⚠️ 최저 반응 (P2) ⓘ
```

| 규칙 | 설명 |
|---|---|
| 최고/최저 세그먼트 자동 하이라이트 | "이 광고의 실질 타깃은 ○○" 판정 문장 생성 |
| 의도 타깃과 실제 반응 타깃 비교 | 기준은 §0의 **사용자 선언 의도**(업종·상품 분류·광고 목적·타깃). 페르소나 반응의 `perceived_target`·`perceived_message` 집계(PERSONA §3.5)와 불일치 시 **타깃 미스매치 경고** — 기획서 1-3 "문제 조기 발견" 가치의 구현 |
| P2 등급 세그먼트 표시 | 40대 이상은 grounding 신뢰도 차등(P2)을 ⓘ로 정직하게 표시 |

### 3-2. 페르소나 보이스 (대표 발화)

**보고서에서 가장 설득력 있는 부분. 정량의 근거를 정성이 증언한다.**

```
┌─ 페르소나 #0341 ─────────────────────────────┐
│ 27세 여성 · 서울 · 직장인 · 개방성↑ 외향성↑     │
│ SNS 헤비유저 (일 2.5h)                        │
│                                              │
│ "썸네일은 진짜 예뻐서 멈췄어요. 근데 끝까지 봐도  │
│  이게 뭘 사라는 건지 모르겠어요. 링크 누를       │
│  이유까진 없달까."                             │
│                                              │
│ Attention ✅ → Interest ✅ → Search ❌         │
└──────────────────────────────────────────────┘
```

| 선정 규칙 | 설명 |
|---|---|
| 3~5명, 의도적으로 다양하게 | 긍정 1 + 부정 1 + 최대 이탈 구간에서 멈춘 페르소나 1 + 타깃/비타깃 각 1 |
| 속성 카드 병기 | 발화자의 데이터 기반 속성(연령·성별·OCEAN 요약·미디어 행동)을 함께 표시 — "지어낸 사람이 아니라 통계에서 샘플링된 사람"임을 보여줌 |
| 퍼널 위치 표시 | 이 페르소나가 AISAS 어디서 멈췄는지 시각화 |
| QA 통과 응답만 | QA 검문소 미통과 발화는 절대 노출 금지 |

## §4. 크리에이티브 진단 — "왜 이 점수인가"

**제작자(디자이너·카피라이터)를 위한 섹션. 점수 → 사유 → 근거의 3단 구조 고정.**

### 진단 항목 (공통 7 + 한국 특화 3)

```
  항목                 점수   업종평균   진단
  ─────────────────────────────────────────────
  훅 (첫 3초)           86     71      ✅ 강한 비주얼 정지력
  메시지 명료성          64     68      ⚠️ 핵심 혜택 전달 불충분
  USP 전달              58     65      ⚠️ "왜 이 제품인가" 부재
  비주얼-카피 정합성      77     70      ✅
  CTA 명확성            41     66      🔴 행동 지시 사실상 부재
  타깃 적합성            71     67      ✅ (단, 실질 타깃 한정)
  브랜드 기억도          69     64      ✅
  ─── 한국 특화 ───────────────────────────────
  체면/과시 소구 적합도   62      —      중립
  트렌드 코드 정합성      74      —      ✅ 현행 코드와 부합 (§6 연동)
  광고 거부감 유발도      33(낮을수록 좋음) —  ✅ 네이티브한 톤
```

### 항목별 상세 카드 양식 (점수 60 미만 항목은 자동 펼침)

```
┌─ 🔴 CTA 명확성  41 / 100  (업종평균 66) ─────────────┐
│                                                     │
│ 진단                                                 │
│  영상 종료 시점에 행동 지시가 없다. "지금 만나보세요"는   │
│  무엇을(앱 설치? 구매? 검색?) 하라는 것인지 불명확.      │
│                                                     │
│ 근거                                                 │
│  · 페르소나 거부/이탈 사유 중 "뭘 하라는지 모름" 31%     │
│  · Search→Action 낙폭 -15%p (업종 평균 낙폭 -9%p)     │
│  · 페르소나 #0341 외 247명이 동일 패턴                 │
│                                                     │
│ → 개선 방향: §5 권고 1 참조                           │
└─────────────────────────────────────────────────────┘
```

| 규칙 | 설명 |
|---|---|
| 점수마다 진단 문장 필수 | 점수만 있는 항목 출력 금지 ("왜"가 없으면 개선 불가) |
| 근거는 §2·§3 데이터를 역참조 | 진단이 페르소나 집계 데이터와 수치로 연결되어야 함 — 공중에 뜬 평가 금지 |
| 한국 특화 항목 | 업종 평균 미보유 시 "—" 표시 (가짜 평균 생성 금지) |

## §5. 개선 권고 — "그래서 뭘 고치나"

**보고서의 종착지. 우선순위가 매겨진 실행 가능한 액션 리스트.**

```
┌─ 개선 권고 (영향도 순) ──────────────────────────────────┐
│                                                         │
│ 1. 🔴 CTA 재설계                       예상 효과: 높음    │
│    현재: "지금 만나보세요" (행동 불명확)                    │
│    제안: 단일 행동 지시로 교체 — 예: "첫 구매 30% 쿠폰 받기" │
│    근거: 거부 사유 31% + Action 낙폭 + CTA 진단 41점       │
│    영향 지표: Action 19% → 업종 평균 26% 수준 회복 기대     │
│                                                         │
│ 2. ⚠️ USP 한 줄 추가                   예상 효과: 중간    │
│    현재: 제품 등장만 있고 차별점 언급 없음                  │
│    제안: 0:07 구간에 핵심 혜택 자막 1줄                    │
│    근거: USP 진단 58점 + Interest→Search 최대 이탈         │
│                                                         │
│ 3. 💡 40대+ 별도 시안 검토              예상 효과: 조건부   │
│    현재 시안은 20-30대 특화. 40대+ 도달이 목표라면          │
│    톤·모델·소구를 달리한 별도 안 권장                       │
│    근거: 세그먼트 히트맵 §3-1 (단, P2 신뢰도 유의)          │
│                                                         │
│ [ 이 권고로 개선안 생성하기 → 광고 생성 ]                   │
└─────────────────────────────────────────────────────────┘
```

| 규칙 | 설명 |
|---|---|
| 최대 3~5개, 영향도 순 | 10개 나열은 0개와 같다. 우선순위가 권고의 본질 |
| 각 권고 = 현재 → 제안 → 근거 → 영향 지표 4요소 고정 | "근거"는 반드시 §2~§4의 데이터 역참조 |
| 예상 효과는 정성 3단계 (높음/중간/조건부) | "개선 시 점수 +12" 같은 **정밀 수치 약속 금지** — 검증 불가능한 단언 |
| 광고 생성 기능 연결 버튼 | 권고 객체가 그대로 4-3 광고 생성 파이프라인의 입력 스키마가 됨 (팀 간 인터페이스) |

## §6. 시장 맥락 — 벤치마크 & 트렌드 (참고 섹션)

**6-A(정량 벤치마크)와 6-B(트렌드 RAG)를 시각적으로도 분리해서 표시.**
벤치마크 DB의 명세·필수 승격·부재 시 강등 규칙은 **ANALYSIS_AGENT_STRATEGY §6-5** 참조.

### 6-A. 업종 벤치마크 비교

```
  지표                  시뮬레이션 (95% CI)    업종 평균 (실측)    비고
  클릭 의향률            16 ~ 22%              CTR 1.6% ⓘ        단위 상이 — 참고 병치
  영상 완주 의향률        34 ~ 45%              38% ⓘ             단위 상이 — 참고 병치
  
  📎 출처: [업종 벤치마크 출처명, 기준 연도] — 정적 DB, 분기 갱신
```

- 출처 없는 벤치마크 표시 금지. 해당 업종 데이터 없으면 섹션 자체를 "벤치마크 데이터 없음"으로 출력.
- **시뮬레이션 지표와 실측 벤치마크는 단위가 다르다** — 동일 스케일처럼 나란히 "판정"하지 않는다. 판정은 A/B 상대 우위 또는 (향후) 업종별 시뮬레이션 기준선 축적 후에만 수행. 실측 평균은 ⓘ 참고 병치만.
- 해당 업종 벤치마크 부재 시, 분석가 토론의 AGT-MS는 결석 처리되며 §4에 "시장 맥락 분석 불가" 명시 (ANALYSIS §6-5 강등 규칙).

### 6-B. 트렌드 소구 코드 (RAG)

```
  현재 [음료] 카테고리 활성 소구 코드          이 광고와의 정합성
  ─────────────────────────────────────────────────
  #제로슈거·헬시플레저                        ✅ 부합 (근거 5건)
  #숏폼 챌린지 연계                           ─ 미사용
  #가심비 (정서적 만족)                       ⚠️ 부분 부합 (근거 3건)
  
  ⓘ 참고용 — 최근 N주 트렌드 문서 RAG 기반. 성과를 보증하지 않음.
```

| 규칙 | 설명 |
|---|---|
| 모든 키워드에 근거 문서 수 표시 | 근거 0건 키워드 출력 금지 (할루시네이션 차단) |
| "참고용" 라벨 고정 | 단언 어조 금지. §4의 "트렌드 코드 정합성" 점수의 출처가 이 섹션 |
| 트렌드 데이터 부족 시 | 빈 섹션 + "트렌드 데이터 부족" 정직 표기 (억지로 채우지 않음) |

## §7. 부록 — 방법론 & 신뢰도 고지

**신뢰는 투명성에서 온다. 접힌 상태 기본, 클릭 시 펼침.**

```
▸ 이 시뮬레이션은 어떻게 작동하나요?
   1,000명의 가상 소비자는 통계청 인구 분포, 서울대-카카오 성격 데이터(N=81만),
   KISDI 미디어패널에서 통계적으로 샘플링되어 생성되었습니다. [파이프라인 도식]

▸ 신뢰도 등급 안내
   · 20-30대 응답: P1 (대규모 한국 데이터 grounding)
   · 40대 이상 응답: P2 (표본 제한 — 참고 수준)
   · 성격 데이터는 자기선택 표본으로 전국 대표성에 한계가 있습니다.

▸ 검증
   본 엔진은 [공개 조사명] 재현 테스트에서 방향성 일치를 확인했습니다. [상세 →]

▸ 면책
   본 보고서는 실제 광고 성과를 보증하지 않으며, 의사결정 참고 자료입니다.
   예측 수치는 가상 소비자 반응 기반 추정 레인지입니다.
```

- PERSONA_STRATEGY §7 금지사항(전국 대표 표본 단언 금지, P2 차등 등)이 사용자 노출 문구에도 그대로 반영되는 지점.
- 검증 데모(§5. 발표의 결정타) 결과를 제품 안에 상시 게시 — 발표용 1회성이 아니라 신뢰 자산화.

| 보고서 섹션 | 데이터 소스 | 생성 주체 |
|---|---|---|
| §0 헤더 | 사용자 입력(소재·세부 정보·타깃) + 감지 의도 교차검증 | 광고 해석 모듈 (ANALYSIS §3.5) |
| §1 결론 카드 | §2+§4 합성 + 목적별 가중치 | 집계 엔진 + 토론 Judge(한 줄 판정 문장만) |
| §2 반응 집계 | 페르소나 응답 구조화 필드 (PERSONA §3.5 스키마, QA 통과분) | 집계 엔진 (LLM 아님) |
| §3 타깃 인사이트 | §2의 세그먼트 분해 + 발화 선정 규칙 + 선언 의도 비교 | 집계 엔진 + 선정 알고리즘 |
| §4 크리에이티브 진단 | 루브릭 평가 패스 점수(토론 전 1회, ANALYSIS §3.5) + §2 역참조 | 점수: 루브릭 평가 패스 / 진단 문장·이견: 분석가 토론 Judge |
| §5 개선 권고 | 토론 확정 claim(진단 근거) × 처방 RAG(처방 근거) | Judge + 이중 근거 검증 (ANALYSIS §6) |
| §6-A 벤치마크 | 정적 벤치마크 DB (ANALYSIS §6-5) | 단순 조회 |
| §6-B 트렌드 | 트렌드 문서 벡터 RAG | RAG + 근거 카운트 |
| §7 부록 | 정적 텍스트 + 엔진 메타데이터(모델 pin + panel_version) | 템플릿 |

### 핵심 구현 원칙
1. **숫자는 집계 엔진이, 문장은 LLM이.** §2의 모든 수치는 LLM이 만들지 않는다 — 페르소나 응답의 구조화 필드를 코드로 집계한다. (PERSONA_STRATEGY 원칙 1의 보고서 버전) **단, 적용 범위는 §2 집계 수치** — §4 차원 점수는 토론 전 루브릭 평가 패스(LLM)가 1회 산출하고 토론은 수정하지 않는다 (ANALYSIS §3.5·원칙 3).
2. **LLM 생성 문장은 전부 데이터 역참조 강제.** 진단·권고 프롬프트에 §2 집계 JSON을 주입하고, 근거 없는 주장 출력을 QA로 필터.
3. **빈 데이터는 빈 채로.** 벤치마크 없음, 트렌드 근거 부족 → 정직하게 "없음" 표시. 억지로 채운 가짜 완성도가 제품을 죽인다.
