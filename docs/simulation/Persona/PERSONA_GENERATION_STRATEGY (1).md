## 0. TL;DR (한 문단 요약)

페르소나의 **다양성은 실제 한국 통계 데이터로 강제 확보**하고, **LLM은 파이프라인의 가장 마지막 단계(서사 생성)에서만 사용**한다. 인구통계·성격·행동 같은 속성(attribute) 생성에 LLM을 쓰면 학습 데이터 평균으로 수렴해 페르소나가 동질화되므로, 속성 샘플링은 전부 LLM **바깥**의 통계 분포에서 수행한다. 성격은 서울대-카카오 OCEAN 데이터(N=813,611)로, 미디어·소비 행동은 KISDI 한국미디어패널로 각각 grounding하며, 두 분포의 결합은 억지 공식 대신 LLM 서사 단계의 추론에 맡긴다. 최종 검증은 "공개된 한국 실제 조사 1건의 방향성 재현"으로 증명한다.

## 1. 배경 및 문제 정의

### 1-1. 우리가 피하려는 실패 모드: 페르소나 동질화

기존 LLM 롤플레이 방식(`"당신은 30대 여성 직장인입니다. 이 광고에 반응하세요"`)은 다음 문제를 가진다.

- LLM은 같은 프롬프트에 대해 학습 데이터의 **평균적 응답**을 반복 생성한다.
- 결과적으로 1,000개의 페르소나를 만들어도 응답이 비슷비슷해진다(변별력 상실).
- 참고 논문 **arXiv:2510.18155 (IEEE ICEBE 2025)** 가 이 한계를 실증한다: 가상 마을 멀티에이전트 시뮬레이션에서 LLM이 환경 밖 응답을 환각하고, 특히 어린이·노인 등 엣지 인구통계의 행동 충실도가 떨어졌다(7세 에이전트가 성인처럼 행동).
  - **이 논문은 "따라야 할 모델"이 아니라 "순수 롤플레이의 한계 증거"로 인용한다.**

### 1-2. 채택하는 방향: Synthetic Data 방식

다양성을 LLM의 상상력이 아니라 **실제 인구 분포에서의 통계적 샘플링**으로 확보한다. 이 방향은 다음 선행 연구가 뒷받침한다.

- **Deepsona** (arXiv 연계, 2025): 6-agent 아키텍처(페르소나 생성 → 노출 → 세그먼트 간 숙의 → 다차원 스코어링 → 품질보증 → 인사이트 종합) + calibration weighting. 실제 조사 재현으로 방향성 정확도 입증.
- **DeepPersona** (arXiv:2511.07338, NeurIPS 2025): taxonomy 기반 progressive conditional sampling. 앵커 트레이트 기준 near/middle/far를 5:3:2 비율로 샘플링해 일관성과 의외성을 균형(동질화 억제).

### 1-3. ClickMe의 차별화 포인트

위 선행 연구·상용 서비스는 모두 **서구/영어권 데이터** 기반이라 한국 소비자의 결이 살아있지 않다. ClickMe는 **한국 실데이터로 grounding된 페르소나**라는 점이 핵심 해자(moat)다.

## 2. 핵심 설계 원칙 (절대 위반 금지)

1. **속성 생성에 LLM을 쓰지 않는다.** 인구통계·성격·행동 등 모든 구조화된 속성은 통계 분포에서 샘플링한다. LLM은 최종 서사 생성 단계에서만 등장한다.
2. **다양성은 데이터가 강제한다.** 실제 한국 인구 비율을 따라 샘플링하므로, 페르소나 집단의 다양성이 구조적으로 보장된다.
3. **억지로 결합하지 않는다.** 데이터가 없는 속성 간 관계는 가짜 숫자로 메우지 말고, "여기까지 데이터 / 이 연결은 추론"이라고 명시적으로 선을 긋는다.
4. **방향성 정확도를 목표로 한다.** "실제 사람과 동일한 응답"이 목표가 아니다. "마케터의 직감보다 나은 의사결정 근거" 제공이 목표다(기획서 핵심 전제).
5. **규모의 함정을 피한다.** 수백만 페르소나는 발표에 불필요하다. 잘 설계된 1,000~10,000 페르소나가 정확도·비용·설득력 모두에서 우월하다.

## 3. 파이프라인 — "조립 라인" 모델

페르소나 1명은 4개 단계(작업대)를 순차 통과하며 생성된다. **단계 1~3은 LLM 없이 통계 샘플링으로만 수행하고, LLM은 단계 4에서 처음 등장한다.**

```
[단계 1]            [단계 2]                  [단계 3]            [단계 4]
인구통계 뼈대   →   성격·행동 채우기      →   한국 특화 속성   →   서사 생성
(통계청 분포)       (카카오 OCEAN +          (한국 소비심리)     (LLM — 여기서 처음)
                    KISDI 미디어패널)
   │                   │                         │                   │
   └─── LLM 사용 안 함 (통계 샘플링만) ──────────┘                   └─ LLM
```

### 단계 1 — 인구통계 뼈대 (Layer 1)

- **무엇을:** 나이·성별·거주지역 등 기본 신상을 정한다.
- **어떻게:** 통계청(KOSIS) 공개 집계표의 실제 한국 인구 비율(주변분포, marginal distribution)을 따라 샘플링한다.
- **왜:** 한국 사회의 실제 다양성이 페르소나 집단에 그대로 복제된다. 다양성을 LLM에 맡기지 않는 첫 번째 방어선.
- **데이터:** KOSIS 공개 집계표(연령×성별×지역). 공개 집계표만으로 충분하다.
  - MDIS 공공용 raw도 무료 다운로드 가능하나(§7.5 정정 참조), 이 단계는 공개 집계표만으로 충분하다.
- **타깃 지정 모드 (`target_filter`) — REPORT_TEMPLATE §0 "타깃 모드" 연동:**
  - 사용자가 타깃(연령대·성별)을 지정해도 **Layer 1을 건너뛰지 않는다.** 연령×성별×지역 결합분포에서 조건에 맞는 셀만 남겨 재정규화한 뒤 샘플링한다(조건부 샘플링).
  - **스킵 금지 이유:** ① Layer 2-α·2-β가 Layer 1의 출력(구체 연령·성별·지역)을 조건 변수로 사용한다 — 스킵하면 조건값이 없다. ② "20대 여성" 안에도 실분포(세부 연령·지역 구성)가 존재한다 — 균등 샘플링은 원칙 2("다양성은 데이터가 강제") 위반. ③ 미지정 속성(지역 등)은 여전히 실분포를 따라야 한다.
  - 타깃이 좁아 표본이 최소선(권장 300명) 미만이면: 보고서에 표본 경고 + 신뢰도 도트 하향 표시 (REPORT §1 신뢰도 규칙 연동).
  - **기본값은 타깃 미지정 = 전체 인구 샘플링.** (hypothesis-discovery 포지셔닝 유지 — 타깃 지정은 사용자가 명시적으로 선택한 경우만)

### 단계 2 — 성격·행동 채우기 (Layer 2, 두 칸으로 분리)

성격과 행동을 **하나의 공식으로 묶지 않고 두 칸으로 나눠** 각각 한국 데이터로 채운다.

#### Layer 2-α — 성격 (자신 있는 칸)
- **무엇을:** OCEAN(Big Five: 개방성·성실성·외향성·친화성·신경증) 성격 차원을 부여한다.
- **어떻게:** 단계 1에서 정해진 연령(과 가능하면 성별)을 조건으로, 서울대-카카오 OCEAN 분포에서 조건부 샘플링한다.
- **데이터:** 서울대 행복연구센터 × 카카오 마음날씨, **N=813,611**, 120문항 Big Five, Nature Scientific Reports 2026-01-13 (DOI: 10.1038/s41598-025-34511-4, Open Access).
  - 연령대별 표본 분포 명시됨(14–19: 162,619 / 20–29: 396,653 / 30–39: 172,623 / 40–49: 54,407 / 50–59: 21,059 / 60+: 4,604).
  - 추출된 5개 성격 유형: Average, Introverted-Open, Introverted-Reactive, Expressive, Conventional → **페르소나 라이브러리 초기 클러스터로 활용 가능.**
- **신뢰도 차등 적용 (중요):**
  - **20~30대: P1 (정식 grounding).** 표본 약 57만 명으로 매우 신뢰 가능.
  - **40대 이상: P2 (추정 + 라벨링 수준).** 50대 이상은 약 3%(25,663명)뿐. BFI-K 고령자 표본(1,038명)으로 보완하거나 보수적 처리.
- **편향 주의:** 카카오 마음날씨를 자발적으로 사용한 사람들의 데이터 → 자기선택 편향(self-selection bias) 존재. 전국 대표 표본이 아님을 코드 주석·리포트에 명시.

#### Layer 2-β — 미디어·소비 행동 (별도 칸)
- **무엇을:** 미디어 이용 시간, 매체별 소비 패턴 등 행동 속성을 부여한다.
- **어떻게:** 단계 1의 인구통계를 조건으로, KISDI 한국미디어패널의 행동 분포에서 조건부 샘플링한다.
- **데이터:** KISDI(정보통신정책연구원) 한국미디어패널조사 — 보고서 단위 교차표(cross-tab)가 상당히 공개됨. 보조: 방통위 방송매체이용행태조사, 언론진흥재단 데이터.

#### ⚠️ Layer 2의 핵심 결정 — 두 칸을 억지로 묶지 말 것
- 카카오 데이터는 **"성격이 어떤지"는 알려주지만 "그 성격이 광고에 어떻게 반응하는지"는 알려주지 않는다.** (성격만 측정, 성격↔광고반응 동시측정 아님)
- 따라서 성격(2-α)과 행동(2-β)을 각각 독립적으로 한국 데이터로 강하게 잡고, **둘 사이의 결합은 단계 4(LLM 서사)의 추론에 맡긴다.**
- **결합 방식 3가지 옵션 (현재 채택: 옵션 2):**
  - 옵션 1 — 서양 학술 문헌의 OCEAN↔행동 매핑 차용 + 한국 적용 신뢰도 감점(weight) 명시. (빠르나 학술적 약점)
  - **옵션 2 (채택) — 성격은 카카오, 행동은 KISDI로 각각 grounding하고, 둘의 연결만 LLM 서사 단계에서 약하게 처리.** (일정 내 현실적 + 발표 시 "양쪽 다 한국 데이터로 grounding" 주장 가능)
  - 옵션 3 — IPF(Iterative Proportional Fitting / raking)로 (연령×OCEAN)과 (연령×행동) 주변분포를 동시 만족하는 결합분포 추정. (가장 정교하나 7/8 일정엔 무리 → **발표 후 Phase로 연기**)

### 단계 3 — 한국 특화 속성 (Layer 2 확장, 차별화 지점)

- **무엇을:** 한국 소비자 특유의 심리 변수를 속성으로 심는다. 예: 체면·타인 시선 의식, 준거집단 동조(주변 따라가기), 눈치, 트렌드/유행 민감도, 가성비·가심비·명품지향·의미소비 같은 소비가치.
- **어떻게:** 한국 소비자 행동 연구·KISDI·대학내일20대연구소·트렌드코리아 시리즈 등에서 추출.
- **왜:** 서구 데이터 기반 경쟁 서비스가 데이터 부재로 흉내 못 내는 부분 → "한국 시장 특화"가 빈말이 아니라 설계에 박힌 차별점이 된다.
- **MBTI 처리:** 성격 근거로는 쓰지 않는다(학술 신뢰도 부족). 단, 결과 리포트에서 **"이 페르소나는 MBTI로 치면 INTJ에 가까움" 같은 표시 라벨**로만 활용(광고주 친숙도 → 커뮤니케이션 도구). 근거 아님, 라벨임.

### 단계 4 — 서사 생성 + 광고 반응 (LLM 최초 등장 — 두 콜로 분리)

단계 4는 **광고와 무관한 콜(4-a)과 광고에 의존하는 콜(4-b)로 분리**한다. 이 분리가 고정 패널(§3.6) 운영과 비용 구조의 전제다.

#### 4-a. 페르소나 프로필/서사 생성 (광고 무관 — 패널 빌드 시 1회)
- **무엇을:** 단계 1~3에서 누적된 **속성 묶음(명세서)**을 LLM에 주입해 일관된 인물 프로필·배경 서사를 생성한다.
- **캐시:** 결과는 페르소나 라이브러리(§3.6)에 저장한다. 시뮬레이션마다 재생성하지 않는다.

#### 4-b. 광고 반응 생성 (광고 의존 — 매 시뮬레이션 N콜)
- **무엇을:** 캐시된 프로필 + 광고(+ KISDI 미디어 다이어리 기반 노출 매체 맥락)를 주입해 "살아있는 한 사람처럼" 광고에 반응하게 만든다.
- **출력:** 반드시 **§3.5 반응 스키마를 준수하는 구조화 출력**이어야 한다 — 자유 서술만으로는 집계 엔진이 동작하지 않는다.
- **캐시 금지:** 반응은 절대 캐시·재사용하지 않는다. 캐시 대상은 프로필(광고 무관)이지 반응(광고 의존)이 아니다.
- **왜 동질화가 안 일어나나:** 1,000개 페르소나 각각이 **서로 다른 실데이터 기반 명세서**를 갖고 있으므로, LLM은 1,000명의 서로 다른 사람을 연기한다. 다양성은 이미 단계 1~3에서 확보됐고 LLM은 그것을 말로 풀어줄 뿐이다.
- **재현성:** 모델 버전을 고정(pin)하고 패널 버전(§3.6)을 함께 기록한다. 모델이 바뀌면 결과 재현성이 깨진다(논문도 모델별 응답 차이를 한계로 지적).

## 3.5. 페르소나 반응 스키마 (팀 내 인터페이스 계약) — 신설

4-b의 출력이자 집계 엔진의 입력. **REPORT_TEMPLATE §2·§3이 요구하는 모든 집계가 이 필드들에서 나와야 한다.** REC 스키마(ANALYSIS §6-2)와 동급의 계약 — 변경 시 집계 엔진·보고서 렌더링 코드와 동시 수정 필수.

```json
{
  "persona_id": "P-000341",
  "ad_id": "AD-1024",
  "exposure_context": "sns_feed_evening",
  "aisas": { "attention": true, "interest": true, "search": false, "action": false, "share": false },
  "drop_stage": "search",
  "drop_reason_tag": "no_reason_to_explore",
  "purchase_intent": 2,
  "trust": 3,
  "rejected": false,
  "rejection_reason_tag": null,
  "emotion_tag": "curiosity",
  "perceived_message": "새로운 음료가 나왔다는 것",
  "perceived_target": "20대 여성",
  "utterance": "썸네일은 예뻐서 멈췄는데 뭘 사라는 건지는 모르겠어요.",
  "qa_passed": true
}
```

| 필드 | 규칙 | 집계 목적지 |
|---|---|---|
| `aisas.*` (bool 5개) | 구조화 필수 | REPORT §2-1 퍼널 |
| `drop_stage` / `drop_reason_tag` | 이탈 지점 + 사유 enum | §2-1 최대 이탈 구간 사유 |
| `purchase_intent` (1~5) / `trust` (1~5) | 정수 제약 필드 | §2-2 히스토그램 / §2-3 |
| `rejection_reason_tag` / `emotion_tag` | **사전 정의 taxonomy enum — 자유 텍스트 금지.** LLM이 직접 enum으로 출력 (사후 분류 아님) | §2-3 거부 분해 / §2-4 감정 분포 |
| `perceived_message` / `perceived_target` | 페르소나가 인식한 메시지·타깃 | §3-1 의도-반응 비교 (타깃 미스매치 판정) |
| `utterance` | 자유 발화 1개 | §3-2 페르소나 보이스 (QA 통과분만 노출) |
| `qa_passed` | QA 검문소 결과 | 집계 포함 여부 게이트 |

> enum 목록(거부 사유·감정·이탈 사유)은 구현 착수 시 REPORT §2-3의 분류 체계와 함께 1회 확정하고, 이후 변경은 집계·렌더링 코드와 동기화한다.

## 3.6. 고정 패널(페르소나 라이브러리) 운영 규칙 — 신설

**채택: 시뮬레이션마다 페르소나를 재생성하지 않고 고정 패널을 운영한다.**

- **결정적 이유 — A/B 비교 타당성:** A안·B안을 서로 다른 랜덤 패널에 노출하면 패널 구성 차이가 광고 차이와 교락(confound)된다. 동일 패널 노출 = 짝지은 비교(paired comparison) → 분산 감소. 기획서 가치 1("A/B 비교 일치율") 측정의 전제 조건.
- **부가 이점:** ① 비용 — 4-a는 패널 빌드 시 1회, 런타임 LLM 비용은 4-b N콜뿐 ② 속도 ③ 사전 QA — 빌드 시점에 모순 프로필 제거 ④ 재현성 — `panel_version` 기록 ⑤ 발표 내러티브 — 실제 리서치 업계의 "고정 패널" 개념과 동일하며, grounding 원천인 KISDI 미디어패널 자체가 동일표본 추적조사다.

| 규칙 | 내용 |
|---|---|
| 기본 패널 | 1,000명, 샘플링 시드 고정, `panel-v1` |
| 타깃 지정 시 | 패널에서 필터링 → 결과가 최소선(300명) 미만이면 `target_filter` 조건부 샘플링으로 증분 생성 후 패널에 추가 (마이너 버전 ↑) |
| 전체 리빌드 | 모델 버전 교체 · grounding 데이터 갱신 시 (메이저 버전 ↑) |
| 보고서 표기 | REPORT §0 엔진 버전 필드에 `panel_version` 병기 |
| 캐시 범위 | **프로필만.** 반응(4-b)은 광고마다 새로 생성 — "응답 재활용" 오해 차단을 위해 보고서 §7 방법론에도 명시 |

## 3.7. 표본·가중(calibration weighting) 운영 규칙 — 신설

**채택: 모집단을 전수 시뮬레이션하지 않고, 통계적으로 충분한 표본을 시뮬레이션한 뒤 가중치로 모집단을 추정한다.** (Deepsona의 population-level aggregation with calibration weighting, 전략서 §6 "규모의 함정")

여론조사가 5,000만 명을 다 묻지 않고 ~1,000명을 묻고 가중치로 전국을 추정하는 것과 같다. 비용(LLM 콜)은 표본 크기로만 결정되고, 결과는 모집단을 대표한다.

### 핵심 원칙 (절대 위반 금지)

1. **표본 크기 ≠ 대표 모집단.** `sample_size`(실제 생성·반응하는 LLM 콜 수)와 "대표하려는 인구"(전국 5,109만 또는 사용자 요청 규모)는 **분리된 두 숫자**다. 비용은 `sample_size`로만 늘고, 대표성은 가중치가 책임진다.
2. **반응을 부풀리지 않는다.** 표본 300명을 시뮬레이션했으면 `페르소나반응` 행은 **정확히 300개**다. 가중치·복제로 1,000행을 만들어내지 않는다 — 없는 응답 합성은 가짜 정밀도이며 §7(동질화·분포 위반)에 해당. **"1,000"은 가중 추정이 대표하는 대상이지 행 수가 아니다.**
3. **각 페르소나는 가중치 `w`를 갖는다** = 그 페르소나가 대표하는 실인구 비율/수. 집계 엔진의 모든 KPI는 **가중 평균**, 신뢰구간은 **가중 부트스트랩**으로 산출한다.
4. **자기가중(self-weighting)이 기본.** 샘플러가 실제 인구분포(연령×성별×지역 등)에서 **비례 추출**하므로, 기본 표본은 가중치가 균일해도 그 자체로 모집단을 대표한다. 가중치가 실제로 일하는 경우는 두 가지뿐:
   - **(a) 타깃 지정** — `target_filter`로 특정 층만 볼 때, 그 층 내부 실분포를 가중으로 유지.
   - **(b) 의도적 과대표집(stratified oversampling)** — 분산이 크거나 표본이 얇은 층(예: 40대+ OCEAN, 희귀 세그먼트)을 일부러 더 뽑아 충실도를 높인 뒤, **과대표집을 가중치로 되돌려** 모집단 비율로 보정. (DeepPersona 5:3:2 stratified의 정량 버전)
5. **유효표본·design effect 정직 표기.** 가중치 편차가 크면 **유효표본수(effective N) < 실제 표본수**다. CI는 유효표본 기준으로 넓어진다. **300명 가중으로 1,000명·전수의 정밀도를 주장하지 않는다.** 보고서는 표본 수·유효표본·대표 모집단을 함께 표기.
6. **최소 표본선 연동.** 표본(또는 셀)이 권장 하한(300) 미만이면 REPORT §1 신뢰도 규칙대로 신뢰도 도트 하향 + 표본 경고.

### 흐름 요약

```
사용자 "1,000명(또는 전국) 기준으로 보고 싶다"
        │  (대표 모집단 = 1,000 / 전국)
        ▼
샘플러: 인구분포 비례로 표본 N명 추출 (예 N=300, self-weighting) + 가중치 w 부여
        │  (LLM 콜 = N = 300, 반응 행 = 300)
        ▼
집계 엔진: 가중 평균 KPI + 가중 부트스트랩 CI  →  "모집단 기준 추정치 ±오차"
        │  (행을 1,000으로 부풀리지 않음 / 유효표본 기준 CI)
```

> **요지:** 클러스터를 대표답 1개로 축약하지 않는다(§7, 분포·이질성 보존). 비용은 **"적게 뽑고 가중으로 키우기"** 로 푼다 — 개별 이질성은 살리고 모집단은 가중이 대표한다.

## 4. 응답 검증 — 두 개의 검문소

LLM이 생성한 반응을 그대로 신뢰하지 않는다. 집계 전에 거른다.

1. **품질보증(QA) 검문소 (필수):** 광고와 무관한 응답, 설정과 모순되는 응답, 앞뒤 안 맞는 응답을 필터링. (Deepsona의 QA Agent에 해당)
2. **다양성 점검 / 숙의(Debate) 검문소 (여유 시):** 반응이 한쪽으로 쏠리면, 다른 관점의 페르소나 간 가벼운 토론으로 응답 분산을 넓힘. (Deepsona의 Debate Agent에 해당) — 필수 아님, 시간 여유 시 추가.

통과한 반응만 집계해 KPI(시뮬레이션 클릭 의향률, 구매의도, 신뢰도, 거부율)로 정리한다. (※ "CTR 예측"이라는 라벨은 calibration 전 사용 금지 — REPORT §1 표기 규칙 참조)

### AISAS 프레임으로 출력 구조화 (권장)
KPI를 AISAS(Attention → Interest → Search → Action → Share) 단계에 매핑하면 보고서 설득력이 올라간다. 검색·공유 단계가 강한 한국 소비 행동에 잘 맞음. (참고 논문도 AIDA→AISAS 소비자 여정 진화를 동기로 인용)

## 5. 검증 데모 — 발표의 결정타

발표에서 반드시 받는 질문: **"가상 소비자 반응이 진짜 사람과 비슷한가?"**

- **방법:** 결과가 이미 공개된 한국 실제 소비자 조사 1건을 골라, 동일 상황을 ClickMe 페르소나에게 동일하게 질의 → **"실제 조사에서 A 우세 → 시뮬레이션도 A 우세"** 방향성 일치를 시연.
- **욕심 금지:** 실제 조사 수치를 소수점까지 맞출 필요 없다. **방향만 맞으면 된다.** (목표가 "동일 응답"이 아니라 "직감보다 나은 근거"이므로)
- **레퍼런스:** Deepsona가 4,834명 USDA 라벨링 연구를 재현해 32~37% 지불의향 프리미엄을 방향성 정확도로 입증한 사례와 동일한 전략.
- **데이터 후보:** KISDI 한국미디어패널 / 방통위 방송매체이용행태조사 / 언론진흥재단 조사 중 결과가 명확한 1건.

## 6. 구현 우선순위 (마감: 2026-07-08, 발표: 2026-07-14)

**원칙: 완벽한 부품이 따로 노는 것보다, 어설퍼도 한 바퀴가 끝까지 도는 게 백 배 낫다.**

### 필수 — 데모 가능 최소선 (먼저 이걸 끝까지 돌린다)
0. **§3.5 반응 스키마 확정** — 집계 엔진·보고서 렌더링과의 선행 계약. 이게 정해져야 4-b·집계를 병렬 착수 가능. (~0.5일)
1. **단계 1** — 행안부/KOSIS 집계표로 연령×성별×지역 샘플링 + `target_filter` 조건부 샘플링. (~1–2일)
2. **단계 2-α** — 카카오 OCEAN 분포로 성격 부여(우선 20~30대만 정식 grounding). (~2일)
3. **단계 2-β** — KISDI 교차표로 행동 분포 부여. (~2일)
4. **단계 4** — 4-a 프로필 생성 → 기본 패널 v1 빌드(1,000명, §3.6) + 4-b 반응 생성(§3.5 스키마 structured output 강제, 모델 버전 고정). (~2–3일)
5. **QA 검문소** — 모순/저품질 응답 필터. (~2일)
6. **기본 가중 집계(§3.7)** — 페르소나 가중치 부여 + 집계 엔진 가중 평균·가중 부트스트랩 CI. 표본+가중으로 비용 통제. (~0.5–1일)
7. **검증 데모** — 한국 실제 조사 1건 방향성 재현. (~2일)

→ 위 1~7이 연결되어 **"페르소나가 광고에 반응하는 한 바퀴"**가 완성되는 것이 데모의 심장.

### 여유 되면
- 단계 3의 한국 특화 심리 속성 확장
- Debate 검문소 / DeepPersona식 5:3:2 stratified sampling(§3.7 과대표집 + 가중 보정)
- AISAS KPI 매핑 리포트

### 발표 후 (Phase 2+)
- 옵션 3(IPF 결합분포 추정)으로 성격↔행동 결합 정교화 + 외부 marginal raking 보정(§3.7 고도화)
- 40대 이상 OCEAN을 BFI-K로 전 연령 확장
- MDIS 주문형 분석으로 풀 상관 매트릭스 확보

- ❌ arXiv:2510.18155를 **따라야 할 모범 모델로 인용하기.** (한계 증거로만 인용)
- ❌ 타깃 지정 시 **Layer 1을 건너뛰기.** (조건부 샘플링으로 통과해야 한다 — Layer 2 조건 변수 + 타깃 내 실분포 유지)
- ❌ A/B 비교를 **서로 다른 랜덤 패널로 수행하기.** (패널 차이가 광고 차이와 교락 — §3.6 고정 패널 사용)
- ❌ 페르소나 **반응(4-b 출력)을 캐시·재사용하기.** (캐시는 프로필(4-a)만 — 반응은 광고마다 새로 생성)
- ❌ 클러스터를 **대표 답변 1개로 축약**해 개별 반응을 대체하기. (분포·세그먼트 이질성 소멸 — §3.7. 비용은 표본+가중으로 푼다)
- ❌ 표본 반응을 **가중치·복제로 부풀려** 표본보다 많은 반응 행 만들기. (가짜 정밀도 — §3.7 원칙2. 행 수 ≠ 대표 모집단)
- ❌ 가중 추정에서 **유효표본 무시하고 전수 정밀도 주장하기.** (design effect로 CI 넓어짐 — §3.7 원칙5)

## 7.5. 데이터 출처 — 단계별 확보처 (구현 시 여기서 받는다)

> **검증 완료:** 단계 1·2-α·2-β는 전부 **공신력 있는 한국 공공·학술 출처에서 무료로** 확보 가능하며, 단계 2-β는 **개인 단위 원시자료(raw)** 까지 받을 수 있다.
> ⚠️ **이전 문서의 정정:** "통계청 마이크로데이터(MDIS)는 200레코드만 받을 수 있다"는 **틀린 정보다.** 200레코드 제한은 다운로드 전 *미리보기 샘플*에만 적용되며, **공공용(public-use) 데이터셋 전체는 회원가입 후 무료로 자유롭게 다운로드** 가능하다. MDIS 제공 원칙 자체가 "제한 없이 제공"이다.

### 단계 1 — 인구통계 뼈대 (Layer 1)
**필요 데이터:** 연령 × 성별 × 지역 인구 비율 (실제 분포)

| 소스 | 제공 내용 | 접근 방법 | 우선순위 |
|---|---|---|---|
| **행정안전부 주민등록 인구통계** (jumin.mois.go.kr) | 행정동 단위까지 성별·연령별 인구수, 매월 갱신 | 웹 다운로드 + 공공데이터포털 파일 | 1순위 (기본 분포) |
| **KOSIS OpenAPI** (kosis.kr) | 시군구/성/연령(5세)별 인구, 프로그램 자동 호출 | REST API (인증키 발급 필요) | 자동화 파이프라인용 |
| **SGIS 통계지리정보 API** (sgisapi.kostat.go.kr) | 총조사 기반 인구, 지도 연계 | REST API | 보조 |

- **구현 메모:** 행안부 주민등록을 기본 분포로 사용(최신성·행정동 단위 강점). 자동화는 KOSIS OpenAPI로 시군구/성/연령 테이블 호출.
- **이 단계는 공개 집계표만으로 100% 해결된다. 별도 신청·비용 없음.**

### 단계 2-α — 성격 (OCEAN)
**필요 데이터:** 한국인 연령대별 Big Five 분포

| 소스 | 제공 내용 | 접근 방법 |
|---|---|---|
| **서울대-카카오 OCEAN** (Nature Sci. Rep. 2026, DOI:10.1038/s41598-025-34511-4) | N=813,611, 연령대별 성격 분포 + 5개 유형 | Open Access 논문 본문·보충자료(supplementary) |
| **BFI-K 고령자 표본** (N=1,038) | 40~60대 보완 | 학술 논문 |

- **구현 메모:** 논문이 Open Access → 본문·supplementary에서 연령대별 분포 수치 추출. 20~30대 정식 grounding(P1), 40대+ BFI-K로 보완(P2).

### 단계 2-β — 미디어·소비 행동 (Layer 2-β) ⭐ 핵심 확보처
**필요 데이터:** 인구통계 조건별 미디어 이용·소비 지출 행동 분포

| 소스 | 제공 내용 | 접근 방법 | 우선순위 |
|---|---|---|---|
| **KISDI 미디어통계포털** (stat.kisdi.re.kr) | 한국미디어패널조사 **원시자료(raw) 무료 다운로드** | 회원가입 후 다운로드 | 1순위 (raw 확보) |
| **방통위 방송매체이용행태조사** | 매체 이용 행태, 교차표 | 미디어통계포털 통합 제공 | 보조 |

- **데이터 규모:** 한국미디어패널조사는 2010년부터 매년 동일표본 추적조사. 2024년 기준 4,006가구·8,693명. 항목: 미디어 기기 보유, 서비스 가입·지출, 미디어 활용, **미디어 다이어리**(하루 매체 이용 시간대 기록).
- **구현 메모:** raw를 받으면 "연령 × 성별 × 미디어 이용 패턴" 교차표를 **직접 계산**해 Layer 2-β를 채운다(집계표 근사 불필요). **미디어 다이어리는 "페르소나가 어떤 매체 맥락에서 광고에 노출되는가" 시뮬레이션에 직접 활용 가능** — 우선 검토할 것.

### 단계 3 — 한국 특화 소비 심리 (Layer 2 확장)
**필요 데이터:** 체면·동조·트렌드 민감도·소비가치(가성비/가심비 등) 분포

| 소스 | 제공 내용 | 접근 방법 | 비고 |
|---|---|---|---|
| **대학내일20대연구소** (20slab.org) | 13가지 소비가치 긍정 응답률, 세대별 소비 성향 | 보고서·인포그래픽 (일부 무료/일부 유료) | 소비가치 1순위 |
| **통계청 사회조사** (MDIS 공공용) | 가치관·생활만족·소비 태도, 개인 단위 raw | MDIS 무료 다운로드 | raw 가능 |
| **한국갤럽 데일리 오피니언** | 사회의식·소비 트렌드 | 일부 공개 / raw는 유료·계약 | 보조 |

- **구현 메모:** 소비가치 분포는 대학내일20대연구소 응답률을 그대로 속성 분포로 사용(예: 성능 87.6%, 품질 80.8%, 편의 71.6% / Z세대는 저렴한 가격 76.0%, 취향·덕질 74.3%). 체면·동조 등 깊은 심리 변수는 통계청 사회조사 가치관 문항(MDIS raw)으로 보완하거나 전문가 사전(prior)으로 보수적 설정.
- **⚠️ 우선순위 주의:** 단계 3은 데이터가 가장 흩어져 있다. **마감 전엔 소비가치 3~4개만 확실히 구현**하고 나머지는 발표 후 확장. 부실하게 많이 넣지 말 것.

### 한눈에 보는 확보 매트릭스

| 단계 | 1순위 소스 | 비용 | raw 확보? | 마감 내 난이도 |
|---|---|---|---|---|
| 1. 인구통계 | 행안부 주민등록 / KOSIS API | 무료 | 집계표 | 쉬움 |
| 2-α. 성격 | 서울대-카카오 OCEAN (Nature) | 무료 | 분포 공개 | 쉬움 |
| 2-β. 행동 | **KISDI 미디어패널 raw** | 무료 | **예** | 중간 |
| 3. 소비심리 | 대학내일20대연구소 + 통계청 사회조사 | 일부 무료 | 일부 | 어려움 |

- **라이선스 메모:** 공공 출처(행안부·통계청·KISDI)는 공공데이터/공공용 마이크로데이터로 분석·활용 허용. 서울대-카카오 OCEAN은 Nature Open Access. 민간 자료(대학내일·한국갤럽)는 공개 수치 인용은 가능하나 **상업적 활용 범위는 서비스화 단계에서 별도 확인 필요**.
