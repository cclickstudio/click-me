# 제너레이터(생성) 기능구현 계획

## 1. 개요

### 목표

사용자의 상품 정보 또는 기존 광고를 기반으로 Meta / Instagram 광고 이미지를 생성한다.

생성된 광고는 사용자가 선택 후 시뮬레이션을 수행할 수 있으며, 시뮬레이션 결과를 기반으로 광고를 개선 생성할 수 있다.

# 7. 광고 후보 정책

광고 후보는 항상 3종 생성한다.

목적은 사용자가 서로 다른 특징과 강조점을 가진 광고를 비교하고 선택할 수 있도록 하는 것이다.

광고안의 순서와 전략은 고정하지 않는다.

AI가 상품, 타겟, 목적을 분석하여 서로 다른 방향의 광고안을 생성한다.

예시

* 광고안 1 : 혜택 강조
* 광고안 2 : 긴급성(FOMO)
* 광고안 3 : 사회적 증거

실제 전략은 상품에 따라 달라질 수 있다.

# 11. 생성 ↔ 시뮬레이터 연동

## 생성 → 시뮬레이터

전달 데이터

* 사용자가 선택한 광고 이미지

별도 메타데이터 저장

* 광고 문구
* 광고 전략
* 템플릿
* 타겟 정보
* 브랜드 정보
* 생성 이유
* QA 결과

현재 버전에서는 시뮬레이터에 광고 이미지만 전달한다.

필요 시 저장된 메타데이터를 활용할 수 있다.

## 시뮬레이터 → 개선생성

전달 데이터

* 개선권고

개선권고 포함 정보

* 진단차원
* 권고등급
* 우선순위
* 권고문장
* 진단근거
* 처방근거

# 12. 고정 패널 정책

패널 = 시뮬레이션에 사용되는 페르소나 집단

정책

* 광고 개선 전/후 비교 시 동일 Panel Version 사용
* 광고 성능 비교 시 동일 Panel Version 사용
* 현재 버전에서는 Panel 변경 불가

예시

광고 A

→ Panel v1

↓

개선권고

↓

광고 A'

→ Panel v1

↓

개선 효과 비교

목적

* 광고만 변경
* 평가 집단은 변경하지 않음

# 13. Generator QA 정책

Generator QA 결과는 광고 생성 메타데이터로 저장한다.

현재 버전에서는

* 시뮬레이터 점수 계산에 사용하지 않음
* 광고 생성 품질 검증 용도로만 사용

저장 대상

* QA 결과
* 검증 로그
* 실패 사유

# 14. 템플릿 정책

## 설계 원칙

템플릿은 단순 레이아웃이 아니라

전략 + 레이아웃

을 함께 정의한다.

예시

Template A

= 제품 강조 전략

* 제품 강조 레이아웃

Template B

= 이벤트 강조 전략

* 이벤트 강조 레이아웃

Template C

= 브랜드 강조 전략

* 브랜드 강조 레이아웃

AI는 상품 정보와 광고 목적을 기반으로 템플릿을 자동 선택한다.
